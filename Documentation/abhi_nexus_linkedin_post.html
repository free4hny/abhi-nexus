How it works — 4 agents, one pipeline
Agent 1 — fetch
Ingests from 20+ sources every 6 hours
Hacker News, TechCrunch, OpenAI Blog, AWS News, Krebs Security, MIT AI News and more. Deduplicates by content hash. Normalizes timestamps to UTC.
feedparser requests pure Python
Agent 2 — rank (the only AI agent)
GPT-4o-mini scores each article 1–100
Batches 10 articles per API call. Assigns importance score, generates plain English summary, produces Hindi translation. Cost: ~$0.003 per full run of 40 articles.
Canvas breach — 275M students affected 90
Critical Windows zero-day patched 88
Minor AWS service update 31
GPT-4o-mini batching structured JSON
Agent 3 — summarize + narrate
English + Hindi audio narration
OpenAI TTS generates clean MP3 audio for every article. Alloy voice for English. Shimmer voice for Hindi. Click and listen — no page reload, no browser voices.
OpenAI TTS-1 English Hindi
Agent 4 — email
Digest delivered at 9 PM, every day
Top 10 articles. Plain English summaries. Scores and tags. HTML email via Gmail SMTP. Scheduler configurable from the UI — no config files needed.
SMTP APScheduler pure Python
What I will not oversell
only 1 agent uses AI
The rank agent uses GPT-4o-mini. Fetch and email are smart Python automation. Knowing when NOT to use AI is as important as knowing when to.
hybrid system
This is not a fully autonomous AI system. It is an autonomous pipeline with one AI-powered step. That was a deliberate choice — reliability over complexity.
4 things building this taught me
1.
Batching saves 75% on API costs
10 articles per call, not 40 separate calls. $2/month instead of $8/month.
2.
Graceful degradation is not optional
One source fails → pipeline continues. OpenAI down → serve cached articles. Never crash on a single failure.
3.
localhost and production are different worlds
Docker networking on Unraid at midnight taught me more than any tutorial.
4.
Separation of concerns matters
The fetch agent knows nothing about ranking. The rank agent knows nothing about email. One job each.
